# Preferred MLIR and LLVM API

**Category:** implementation
**Enforcement:** mandatory — use the modern, non-deprecated API for all new and modified code

## Rule

Always use the recommended MLIR and LLVM API forms. The deprecated forms produce compiler warnings and will be removed in future versions. When modifying existing code that uses a deprecated API, update to the recommended form as part of the change.

---

## MLIR API

### OpBuilder::create<T>() → T::create()

The `OpBuilder::create<T>()` template method is deprecated. Use the static `::create()` method on the Op class instead.

This applies to all builder variable names — `builder`, `rewriter`, `b`, `modBuilder`, or any other `OpBuilder` / `PatternRewriter` instance.

```cpp
// ❌ Deprecated
Value result = builder.create<arith::MulIOp>(loc, lhs, rhs);
auto op = rewriter.create<linalg::GenericOp>(loc, types, inputs, outputs, ...);
Value cst = b.create<arith::ConstantOp>(loc, type, attr);

// ✅ Recommended
Value result = arith::MulIOp::create(builder, loc, lhs, rhs);
auto op = linalg::GenericOp::create(rewriter, loc, types, inputs, outputs, ...);
Value cst = arith::ConstantOp::create(b, loc, type, attr);
```

Multi-line and chained calls follow the same pattern:

```cpp
// ❌ Deprecated
Value v = builder
              .create<linalg::FillOp>(loc, inputs, outputs)
              .getResult(0);

// ✅ Recommended
Value v = linalg::FillOp::create(builder, loc, inputs, outputs)
              .getResult(0);
```

The static `::create()` method is generated by MLIR tablegen for every Op and takes `OpBuilder &` as its first argument.

### Type::cast<T>() → mlir::cast<T>(type) / isa<T>(type) / dyn_cast<T>(type)

```cpp
// ❌ Deprecated
auto shapedType = val.getType().cast<ShapedType>();
if (val.getType().isa<FunctionType>()) { ... }
auto funcType = val.getType().dyn_cast<FunctionType>();

// ✅ Recommended
auto shapedType = mlir::cast<ShapedType>(val.getType());
if (isa<FunctionType>(val.getType())) { ... }
auto funcType = dyn_cast<FunctionType>(val.getType());
```

## LLVM API

### PointerUnion::is<T>() → isa<T>(val)

`llvm::PointerUnion` (and MLIR's `OpFoldResult` which wraps it) deprecated the `.is<T>()` member accessor.

```cpp
// ❌ Deprecated
if (ub.is<Attribute>()) { ... }

// ✅ Recommended
if (isa<Attribute>(ub)) { ... }
```

### PointerUnion::get<T>() → cast<T>(val)

```cpp
// ❌ Deprecated
auto attr = ub.get<Attribute>();

// ✅ Recommended
auto attr = cast<Attribute>(ub);
```

### llvm::cast<T>() / isa<T>() / dyn_cast<T>()

These lightweight RTTI alternatives are the preferred way to cast LLVM/MLIR value types. They are NOT deprecated and should be used instead of C++ `dynamic_cast` or `static_cast` for LLVM/MLIR types.

```cpp
// ✅ Always preferred for LLVM/MLIR types
auto intAttr = cast<IntegerAttr>(attr);
if (auto forOp = dyn_cast<scf::ForOp>(op)) { ... }
if (isa<arith::ConstantOp>(op)) { ... }
```

## Summary Table

| Deprecated API | Recommended API | Domain |
|---|---|---|
| `builder.create<Op>(loc, args...)` | `Op::create(builder, loc, args...)` | MLIR |
| `rewriter.create<Op>(loc, args...)` | `Op::create(rewriter, loc, args...)` | MLIR |
| `b.create<Op>(loc, args...)` | `Op::create(b, loc, args...)` | MLIR |
| `type.cast<T>()` | `mlir::cast<T>(type)` | MLIR |
| `type.isa<T>()` | `isa<T>(type)` | MLIR |
| `type.dyn_cast<T>()` | `dyn_cast<T>(type)` | MLIR |
| `val.is<T>()` | `isa<T>(val)` | LLVM (PointerUnion) |
| `val.get<T>()` | `cast<T>(val)` | LLVM (PointerUnion) |

## Audit Commands

Run these before committing to catch deprecated API usage:

```bash
# Deprecated OpBuilder::create<T>()
grep -rn '\.create<' --include='*.cpp' --include='*.h' .

# Deprecated PointerUnion::is<T>() / get<T>()
grep -rn '\.is<[^>]*>()' --include='*.cpp' --include='*.h' .
grep -rn '\.get<[^>]*>()' --include='*.cpp' --include='*.h' .

# Deprecated Type::cast<T>() / isa<T>() / dyn_cast<T>()
grep -rn '\.cast<[^>]*>()' --include='*.cpp' --include='*.h' .
grep -rn '\.isa<[^>]*>()' --include='*.cpp' --include='*.h' .
grep -rn '\.dyn_cast<[^>]*>()' --include='*.cpp' --include='*.h' .
```

Exclude generated `.inc` files from all audit results.

## Process

1. Write new code using only the recommended API forms
2. When touching a line that uses a deprecated form, update it in the same commit
3. The project compiles with `-Werror=deprecated-declarations` — any deprecated API usage is a **hard build error**, not a warning
4. Build with `ninja` and resolve all errors before committing
5. Run the audit commands above before finalizing a PR
